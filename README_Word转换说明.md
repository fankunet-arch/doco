## 🎨 应用的专业格式标准

### 文档结构
每个 Word 文档都包含：
- ✅ **封面页** - 精美的标题页，包含文档标题、装饰线和生成日期
- ✅ **目录页** - 自动生成的可点击目录（1-3级标题）
- ✅ **正文内容** - 完整格式化的文档内容
- ✅ **页码** - 页脚右下角显示页码

### 页面设置
- **纸张大小**: A4 (21cm × 29.7cm)
- **页边距**:
  - 上/下: 2.54cm
  - 左/右: 3.17cm（标准办公文档边距）

### 字体与样式

#### 正文
- 字体：Microsoft YaHei（微软雅黑）
- 字号：11pt
- 行距：1.15倍行距
- 段后间距：8pt

#### 标题层级
- **一级标题 (Heading 1)**
  - 颜色：深蓝色 (#2E74B5)
  - 字号：16pt
  - 样式：加粗
  - 段前/段后：18pt / 12pt

- **二级标题 (Heading 2)**
  - 颜色：黑色
  - 字号：14pt
  - 样式：加粗
  - 段前/段后：14pt / 10pt

- **三级标题 (Heading 3)**
  - 颜色：黑色
  - 字号：12pt
  - 样式：加粗
  - 段前/段后：12pt / 8pt

#### 特殊格式

1. **代码块**
   - 字体：Consolas（等宽字体）
   - 字号：9pt
   - 背景色：浅灰色 (#F5F5F5)
   - 左缩进：0.5cm
   - 保持代码原始缩进

2. **引用块 (Blockquotes)**
   - 字体：Microsoft YaHei
   - 样式：斜体
   - 颜色：深灰色 (#505050)
   - 左侧边框：深蓝色竖线
   - 左缩进：1cm

3. **表格**
   - 样式：Light Grid Accent 1（浅色网格）
   - 表头：加粗显示
   - 边框：清晰的网格线

4. **内联格式**
   - **粗体文本**: 使用加粗字体
   - `行内代码`: Consolas字体，红色高亮 (#C7254E)

## 🔧 技术实现

### 使用的工具
- **Python 3** - 核心处理引擎
- **python-docx** - Word 文档生成库
- **正则表达式** - Markdown 解析

### 转换脚本
主程序：`md_to_word_converter.py`

#### 重新生成文档
如需重新生成所有 Word 文档，运行：

```bash
python md_to_word_converter.py
```

#### 清理并重新生成
```bash
rm -rf word/
python md_to_word_converter.py
cd word && zip -r ../软冰项目标准化文档集_Word版.zip *.docx
```

## 📋 使用建议

### 打开文档后的操作

1. **更新目录**
   - 在 Word 中打开文档后，右键点击目录
   - 选择"更新域" → "更新整个目录"
   - 这样可以确保目录页码完全准确

2. **检查格式**
   - 所有格式已预设完成
   - 如需调整，可使用 Word 的"样式"功能统一修改

3. **导出 PDF**
   - 文件 → 另存为 → PDF
   - 保持原有格式和目录链接

### 适用场景
- ✅ 公司内部培训文档
- ✅ 技术标准化手册
- ✅ 产品操作指南
- ✅ 项目设计文档
- ✅ 需要打印的技术资料

## 📝 注意事项

1. **字体兼容性**
   - Microsoft YaHei（微软雅黑）是 Windows 系统标准字体
   - macOS 用户可能需要安装此字体，或使用"苹方"替代

2. **目录功能**
   - 目录是通过 Word 域代码实现的
   - 首次打开可能需要手动更新（右键目录 → 更新域）

3. **表格处理**
   - Markdown 表格已转换为 Word 表格
   - 复杂表格可能需要手动微调对齐

4. **代码块**
   - 保持了原始代码的缩进
   - 使用等宽字体和灰色背景便于识别
